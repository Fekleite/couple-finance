# Data Model: F10 - Auditoria simples de alteracoes financeiras

## Audit Event

**Table**: `public.financial_audit_events`

Registro visivel ao usuario de uma alteracao financeira importante e salva em
transacao ou meta autorizada.

### Fields

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id` | `uuid` | Yes | Primary key, generated by database |
| `occurred_at` | `timestamptz` | Yes | Instante da acao, default `now()` |
| `actor_user_id` | `uuid` | No | Usuario autenticado que realizou a acao; nulo somente para autoria indisponivel |
| `item_type` | `text` | Yes | `transaction` ou `goal` |
| `item_id` | `uuid` | Yes | ID do item auditado |
| `action_type` | `text` | Yes | `created`, `updated`, `completed`, `archived`, `removed_from_main_flow` |
| `visibility` | `text` | Yes | `individual` ou `shared` |
| `owner_user_id` | `uuid` | Conditional | Obrigatorio para eventos individuais |
| `shared_budget_id` | `uuid` | Conditional | Obrigatorio para eventos compartilhados |
| `subject_label` | `text` | Yes | Nome/titulo normalizado do item, 1-120 caracteres |
| `subject_amount_cents` | `bigint` | No | Valor autorizado principal, quando aplicavel |
| `subject_date` | `date` | No | Data civil autorizada, quando aplicavel |
| `subject_status` | `text` | No | Status autorizado, principalmente para metas |
| `summary_key` | `text` | Yes | Chave fechada para mensagem segura de apresentacao |
| `created_at` | `timestamptz` | Yes | Default `now()` |

### Relationships

- `actor_user_id` referencia `auth.users(id)` com `on delete set null`, para
  preservar o evento com fallback de autoria indisponivel.
- `owner_user_id` referencia `auth.users(id)` para eventos individuais.
- `shared_budget_id` referencia `public.shared_budgets(id)` para eventos
  compartilhados.
- `item_id` referencia conceitualmente `public.financial_transactions(id)` ou
  `public.financial_goals(id)`, mas nao usa FK polimorfica direta.

### Constraints

- `item_type in ('transaction', 'goal')`.
- `action_type in ('created', 'updated', 'completed', 'archived',
  'removed_from_main_flow')`.
- `visibility in ('individual', 'shared')`.
- Individual exige `owner_user_id is not null` e `shared_budget_id is null`.
- Shared exige `shared_budget_id is not null` e `owner_user_id is null`.
- `subject_label = btrim(subject_label)` e tamanho entre 1 e 120 caracteres.
- `subject_amount_cents` nulo ou entre 1 e `99999999999`.
- `summary_key` pertence a conjunto fechado documentado no contrato de
  apresentacao.
- Eventos nao podem existir sem escopo de autorizacao claro.

### Authorization

- RLS habilitado.
- Eventos individuais: leitura somente quando `owner_user_id = auth.uid()`.
- Eventos compartilhados: leitura somente para membros ativos do mesmo
  `shared_budget_id`.
- Convites pendentes ou encerrados nao concedem acesso.
- Revogacao de membership remove acesso na proxima consulta.
- Clientes autenticados nao recebem `INSERT`, `UPDATE` ou `DELETE` diretos.

### Indexes

- `(owner_user_id, occurred_at desc, id desc)` para eventos individuais.
- `(shared_budget_id, occurred_at desc, id desc)` para eventos compartilhados.
- `(item_type, item_id, occurred_at desc)` para validacao e troubleshooting
  autorizado em testes.
- `(actor_user_id, occurred_at desc)` somente se testes mostrarem necessidade;
  nao deve sustentar ranking ou comparacao.

## Audit Action Type

| Value | Meaning | Applies to |
|-------|---------|------------|
| `created` | Item financeiro criado | Transacao, meta |
| `updated` | Campos financeiros/contextuais relevantes editados | Transacao, meta |
| `completed` | Meta concluida | Meta |
| `archived` | Meta arquivada | Meta |
| `removed_from_main_flow` | Item retirado do fluxo principal, se acao existir | Transacao |

Abrir tela, filtrar, ordenar, cancelar formulario e interacoes sem salvamento
nao geram eventos.

## Audited Item

Representacao polimorfica do item alterado.

| Field | Type | Rule |
|-------|------|------|
| `itemType` | `transaction \| goal` | Define rotulo e campos esperados |
| `itemId` | `string` | UUID do item original |
| `label` | `string` | Titulo da transacao ou nome da meta |
| `amountCents` | `number \| null` | Valor da transacao ou valor alvo/atual conforme mensagem |
| `date` | `YYYY-MM-DD \| null` | Data da transacao ou prazo da meta quando seguro |
| `status` | `string \| null` | Status da meta quando relevante |

## Audit Visibility Scope

| Value | Storage rule | Access rule |
|-------|--------------|-------------|
| `individual` | `owner_user_id` preenchido e `shared_budget_id` nulo | Somente pessoa proprietaria |
| `shared` | `shared_budget_id` preenchido e `owner_user_id` nulo | Membros ativos do espaco compartilhado |

Autoria nao concede acesso. Responsabilidade financeira nao concede acesso por
si so.

## Audit Actor

Pessoa que realizou a acao auditada.

### Rules

- Derivada de `auth.uid()` dentro da mutacao financeira.
- Nao aceita ID confiavel enviado pelo cliente.
- Exibida como `Voce` quando corresponde ao usuario atual.
- Exibida como `Pessoa parceira` quando o evento compartilhado pertence ao
  casal ativo e a identidade especifica nao estiver disponivel.
- Exibida como `Autoria indisponivel` quando nao puder ser mostrada com
  seguranca.
- Nunca usa `user_metadata`, e-mail ou nome enviado pelo cliente como base de
  autorizacao.

## Audit Safe Snapshot

Contexto minimo persistido para explicar o evento sem consultar dados
inacessiveis.

### Rules

- Snapshot contem somente dados que a pessoa autorizada poderia ver no item no
  momento do evento.
- Para transacoes: titulo, valor, data civil e visibilidade sao suficientes
  para o MVP.
- Para metas: nome, valor alvo ou atual quando relevante, prazo/status e
  visibilidade sao suficientes para o MVP.
- Edicoes usam snapshot atual e mensagem simples; nao persistem diff completo.
- Snapshot nao inclui observacoes longas, detalhes internos de permissao,
  SQL, IDs sensiveis desnecessarios, e-mail ou dados individuais de outra
  pessoa.

## Authorized Audit Query

Consulta de listagem executada contra `public.financial_audit_events` sob RLS.

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `limit` | `number` | No | Default 50; maximo 100 |
| `itemType` | `transaction \| goal \| all` | No | Default `all`; filtro simples opcional |
| `visibility` | `individual \| shared \| all` | No | Default `all`; filtro simples opcional |

### Result

| Field | Type | Meaning |
|-------|------|---------|
| `items` | `AuthorizedAuditEvent[]` | Eventos autorizados recentes |
| `hasActiveSharedBudget` | `boolean` | Contexto de relacionamento autorizado |
| `generatedAt` | `string` | Momento da leitura no cliente |

Nao ha contagem total no MVP.

## Audit Presentation State

| State | Meaning | Safe retained data |
|-------|---------|--------------------|
| `loading` | Primeira leitura ou contexto mudou | Nenhum evento antigo |
| `ready` | Lista autorizada carregada | Eventos autorizados recentes |
| `empty` | Nenhum evento autorizado retornado | Mensagem segura |
| `refreshing` | Revalidacao em andamento | Lista atual se sessao/contexto nao mudou |
| `error` | Falha recuperavel | Nenhum detalhe privado |
| `blocked` | Usuario sem sessao valida | Mensagem segura e rota protegida |

Somente a requisicao mais recente pode transicionar para `ready`, `empty` ou
`error`.

## Shared Audit Authorization

### Rules

- Pessoa sem budget ativo pode consultar eventos individuais proprios.
- Pessoa sem budget ativo nao ve eventos compartilhados nem mensagens que
  sugiram sua existencia.
- Pessoa com budget ativo ve eventos compartilhados desse budget e eventos
  individuais proprios.
- Membership removida bloqueia leitura compartilhada na proxima consulta.
- Autor que nao pertence mais ao casal pode continuar aparecendo apenas por
  fallback seguro, sem reabrir acesso a dados privados.
