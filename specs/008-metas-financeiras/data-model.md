# Data Model: F09 - Metas financeiras

## Financial Goal

**Table**: `public.financial_goals`

Objetivo financeiro acompanhado manualmente por uma pessoa ou por membros ativos
de um espaco financeiro compartilhado.

### Fields

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id` | `uuid` | Yes | Primary key, generated by database |
| `name` | `text` | Yes | Normalizado, 1-80 caracteres |
| `target_amount_cents` | `bigint` | Yes | 1 a `99999999999` |
| `current_amount_cents` | `bigint` | Yes | 0 a `99999999999` |
| `deadline_date` | `date` | No | Data civil opcional `YYYY-MM-DD` |
| `visibility` | `text` | Yes | `individual` ou `shared` |
| `status` | `text` | Yes | `active`, `completed` ou `archived` |
| `created_by_user_id` | `uuid` | Yes | Derivado de `auth.uid()` |
| `shared_budget_id` | `uuid` | Conditional | Obrigatorio somente para shared |
| `completed_at` | `timestamptz` | No | Preenchido ao concluir |
| `archived_at` | `timestamptz` | No | Preenchido ao arquivar |
| `created_at` | `timestamptz` | Yes | Default `now()` |
| `updated_at` | `timestamptz` | Yes | Atualizado por trigger |

### Relationships

- `created_by_user_id` referencia `auth.users(id)` com exclusao restrita.
- `shared_budget_id` referencia `public.shared_budgets(id)` com exclusao
  restrita.
- Metas compartilhadas dependem de `public.budget_members` ativo para leitura e
  mutacao, mas nao armazenam membros individualmente.

### Constraints

- `name = btrim(name)` e tamanho entre 1 e 80 caracteres.
- `target_amount_cents > 0` e `target_amount_cents <= 99999999999`.
- `current_amount_cents >= 0` e `current_amount_cents <= 99999999999`.
- `visibility in ('individual', 'shared')`.
- `status in ('active', 'completed', 'archived')`.
- Individual exige `shared_budget_id is null`.
- Shared exige `shared_budget_id is not null`.
- `completed_at` deve existir quando `status = 'completed'`.
- `archived_at` deve existir quando `status = 'archived'`.
- Visibilidade e `shared_budget_id` nao sao alterados apos criacao no MVP.

### Authorization

- RLS habilitado.
- Individual: leitura e mutacao somente para `created_by_user_id = auth.uid()`.
- Shared: leitura e mutacao somente para membros ativos do mesmo
  `shared_budget_id`.
- Convites pendentes ou estados terminais nao concedem acesso.
- Membership removida ou budget inativo remove acesso na proxima consulta.
- Metas inacessiveis nao retornam em listas, detalhes, estados, erros ou
  resumos.

### Indexes

- `(created_by_user_id, status, updated_at desc, id desc)` para metas
  individuais.
- `(shared_budget_id, status, updated_at desc, id desc)` para metas
  compartilhadas.
- Indices parciais equivalentes para `status = 'active'` se a analise de
  migration mostrar ganho real.
- `(deadline_date, id)` para ordenacao de metas ativas com prazo.

## Goal Visibility

| Value | Storage rule | Access rule |
|-------|--------------|-------------|
| `individual` | `shared_budget_id is null` | Somente pessoa criadora |
| `shared` | `shared_budget_id is not null` | Membros ativos do espaco compartilhado |

Visibilidade define acesso. Ela nao pode ser alterada apos criacao no MVP.

## Goal Status

| Value | Meaning | List behavior |
|-------|---------|---------------|
| `active` | Meta em acompanhamento | Aparece na lista principal |
| `completed` | Meta encerrada como atingida | Sai da lista ativa e fica em historico simples |
| `archived` | Meta retirada do acompanhamento | Sai da lista ativa e fica separada de concluidas |

### State transitions

```text
active -> completed
active -> archived
completed -> archived
archived -> active (optional only if explicitly supported by tasks)
completed -> active (optional only if explicitly supported by tasks)
```

O plano prioriza `active -> completed` e `active -> archived`. Reabrir metas e
opcional para tasks se nao aumentar escopo.

| Field | Type | Required | Rules |

| `visibility` | `individual \| shared \| all` | No | Default `all` |

| Field | Type | Meaning |

Nao ha contagem total no MVP.

## Goal Mutation

Payload canonico para criar ou editar uma meta.

| Field | Type | Notes |
|-------|------|-------|
| `name` | `string` | Normalizado no frontend e banco |
| `targetAmountCents` | `number` | Inteiro positivo |
| `currentAmountCents` | `number` | Inteiro nao negativo |
| `deadlineDate` | `YYYY-MM-DD \| null` | Data civil opcional |
| `visibility` | `individual \| shared` | Usado somente na criacao |
| `sharedBudgetId` | `string \| null` | Obrigatorio para shared |

### Mutation rules

- Criacao individual ignora qualquer `sharedBudgetId` enviado.
- Criacao compartilhada exige membership ativa no budget informado.
- Edicao nao aceita alterar `visibility` ou `sharedBudgetId`.
- Edicao de meta concluida ou arquivada deve ser bloqueada no MVP, salvo se
  tasks decidirem permitir reabertura explicitamente.
- Concluir define `status = 'completed'` e `completed_at = now()`.
- Arquivar define `status = 'archived'` e `archived_at = now()`.

## Goal Progress

Modelo derivado, nao persistido.

| Field | Type | Rule |
|-------|------|------|
| `percent` | `number` | `currentAmountCents / targetAmountCents * 100` |
| `displayPercent` | `number` | Percentual arredondado para exibicao |
| `barPercent` | `number` | Limitado entre 0 e 100 |
| `remainingAmountCents` | `number` | `max(target - current, 0)` |
| `exceededAmountCents` | `number` | `max(current - target, 0)` |
| `achievement` | `in_progress \| reached \| exceeded` | Derivado dos valores |

## Goal Deadline State

| State | Rule | Message intent |
|-------|------|----------------|
| `none` | `deadlineDate is null` | Meta sem prazo |
| `future` | Prazo no futuro e meta ativa | Dentro do prazo |
| `today` | Prazo igual a hoje civil | Prazo hoje |
| `overdue` | Prazo passou e meta nao atingida | Linguagem neutra |
| `completed` | Meta concluida | Concluida, independentemente do prazo |

Comparacoes usam data civil local normalizada sem converter o valor persistido
para timestamp financeiro.

## Goal Presentation State

| State | Meaning | Allowed retained data |
|-------|---------|-----------------------|
| `loading` | Primeira leitura ou contexto mudou | Nenhuma meta antiga |
| `ready` | Lista autorizada carregada | Itens e filtros atuais |
| `empty` | Nenhuma meta autorizada no filtro | Mensagem segura |
| `submitting` | Criacao, edicao ou transicao em andamento | Payload atual e lista autorizada se contexto nao mudou |
| `success` | Mutacao concluiu e lista foi revalidada | Mensagem curta |
| `error` | Falha recuperavel | Nenhum detalhe privado |
| `blocked` | Contexto impede acao compartilhada | Mensagem segura |

Somente a requisicao mais recente pode transicionar para `ready`, `empty` ou
`error`.

## Shared Goal Authorization

Contexto derivado de `public.budget_members` e `shared_budgets`.

### Rules

- Pessoa sem budget ativo pode criar e listar metas individuais.
- Pessoa sem budget ativo nao pode criar meta compartilhada.
- Pessoa com budget ativo pode criar meta compartilhada para esse budget.
- Ambos os membros ativos podem ler e mutar metas compartilhadas desse budget.
- Membership removida bloqueia nova leitura e mutacao compartilhada.
- Mensagens nao diferenciam meta inexistente de meta inacessivel.
