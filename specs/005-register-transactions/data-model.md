# Data Model: F05 - Registro de transacoes

## Financial Transaction

**Table**: `public.financial_transactions`

Registro imutavel nesta feature que representa uma receita ou despesa
individual ou compartilhada.

### Fields

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id` | `uuid` | Yes | Primary key, generated by database |
| `title` | `text` | Yes | Normalizado, 1-120 caracteres |
| `amount_cents` | `bigint` | Yes | 1 a `99999999999` |
| `transaction_type` | `text` | Yes | `income` ou `expense` |
| `transaction_date` | `date` | Yes | Dia civil `YYYY-MM-DD` |
| `category_code` | `text` | Yes | FK para categoria padrao |
| `created_by_user_id` | `uuid` | Yes | Derivado de `auth.uid()` |
| `responsible_user_id` | `uuid` | Yes | Criador ou membro ativo elegivel |
| `visibility` | `text` | Yes | `individual` ou `shared` |
| `shared_budget_id` | `uuid` | Conditional | Obrigatorio somente para shared |
| `observation` | `text` | No | Nulo ou normalizado, ate 500 caracteres |
| `idempotency_key` | `uuid` | Yes | Unica por criador e tentativa |
| `created_at` | `timestamptz` | Yes | Default `now()` |

### Relationships

- `category_code` references
  `public.standard_financial_categories(code)` com exclusao restrita.
- `created_by_user_id` e `responsible_user_id` referenciam `auth.users(id)` com
  exclusao restrita.
- `shared_budget_id` referencia `public.shared_budgets(id)` com exclusao
  restrita.

### Constraints

- Titulo e observacao usam `btrim`; titulo nao pode ficar vazio.
- `amount_cents > 0` e `amount_cents <= 99999999999`.
- Tipo aceita apenas `income` ou `expense`.
- Visibilidade aceita apenas `individual` ou `shared`.
- Individual exige `shared_budget_id is null` e
  `created_by_user_id = responsible_user_id`.
- Shared exige `shared_budget_id is not null`.
- `(created_by_user_id, idempotency_key)` e unico.
- Categoria ativa/aplicavel e memberships ativas sao validadas na operacao de
  criacao, pois dependem de outras tabelas e estado corrente.

### Authorization

- RLS habilitado.
- Individual: `SELECT` somente para `created_by_user_id = auth.uid()`.
- Shared: `SELECT` somente para membership ativa no `shared_budget_id`.
- Sem policy ou grant de `INSERT`, `UPDATE` ou `DELETE` para clientes.
- Criacao somente pela RPC autenticada.

### Indexes

- Unique `(created_by_user_id, idempotency_key)`.
- `(created_by_user_id, transaction_date desc)`.
- `(shared_budget_id, transaction_date desc)` com predicado para linhas shared.

## Transaction Visibility

| Value | Storage rule | Access rule |
|-------|--------------|-------------|
| `individual` | Sem `shared_budget_id`; criador e responsavel iguais | Somente criador |
| `shared` | Com `shared_budget_id`; criador e responsavel ativos no espaco na criacao | Membros ativos do espaco |

Visibilidade define acesso. Responsabilidade nao concede acesso por si so.

## Transaction Creator

Pessoa autenticada que confirma o registro.

### Rules

- Sempre derivada de `auth.uid()` no banco.
- Nunca aceita como campo confiavel do cliente.
- Permanece distinta do responsavel em transacoes compartilhadas.
- Participa da chave de idempotencia.

## Responsible Person

Pessoa associada a responsabilidade da transacao.

### Rules

- Individual: sempre o criador.
- Shared: membro ativo do mesmo espaco no momento da confirmacao.
- Deixar de ser membro antes da confirmacao bloqueia a criacao.
- Nao determina autorizacao de leitura fora das regras de visibilidade.

## Standard Category Reference

Referencia a categoria canonica da F04.

### Rules

- Usa `category_code`, nunca nome exibido.
- Deve existir, estar ativa e ser aplicavel ao tipo ou `both` na confirmacao.
- Categoria invalida nao e convertida para `other`.
- Categoria inativada depois da criacao preserva a referencia historica.

## Shared Financial Space Reference

Referencia ao `public.shared_budgets` criado na F02.

### Rules

- Ausente para individual.
- Obrigatoria e ativa para shared.
- Criador e responsavel devem possuir memberships ativas no mesmo espaco.
- Convite pendente ou estado terminal nao concede elegibilidade.

## Transaction Submission

Payload canonico enviado a RPC.

| Field | Type | Notes |
|-------|------|-------|
| `title` | `string` | Normalizado pelo frontend e banco |
| `amountCents` | `number` | Inteiro exato |
| `transactionType` | `income \| expense` | Define aplicabilidade |
| `transactionDate` | `YYYY-MM-DD` | Data civil |
| `categoryCode` | `string` | Categoria selecionada |
| `responsibleUserId` | `uuid \| null` | Ignorado/substituido pelo criador em individual |
| `visibility` | `individual \| shared` | Escopo explicito |
| `sharedBudgetId` | `uuid \| null` | Obrigatorio em shared |
| `observation` | `string \| null` | Vazio normaliza para nulo |
| `idempotencyKey` | `uuid` | Preservada em retry |

## Idempotency Key

Identidade da tentativa de confirmacao.

### Lifecycle

| Trigger | Current | Result |
|---------|---------|--------|
| Formulario fica pronto | absent | Nova UUID |
| Envio ou retry recuperavel | active | Mesma UUID |
| Payload editado apos tentativa | active | Nova UUID antes do proximo envio |
| Mesmo payload e mesma UUID | persisted | Retorna mesma transacao |
| Payload diferente e mesma UUID | persisted | Conflito seguro, sem nova linha |
| Registrar outra transacao | persisted | Nova UUID e novo formulario |

## Submission State

| State | Meaning | Safe retained data |
|-------|---------|--------------------|
| `loading_context` | Categorias ou relacionamento em verificacao | Nenhuma opcao privada parcial |
| `ready` | Formulario editavel | Todos os campos validos |
| `submitting` | RPC em andamento | Payload e chave atuais |
| `recoverable_error` | Retry permitido | Textos, valor, tipo e data; opcoes revalidadas |
| `blocked` | Autorizacao/contexto invalido | Somente mensagem segura |
| `success` | Transacao criada ou recuperada idempotentemente | Resumo autorizado |
